Considering Airbnb's immense success during its 9 years of service, it was no wonder that Ferrosia started receiving applications for Airbnb integration.

BI-DIRECTIONAL COMMUNICATION BETWEEN NATIVE SITE AND AIRBNB

In general, one customer planned to maximize their access to rental property by establishing a direct line of bi-directional contact between the pre-existing site of the company and Airbnb.

Ferrosia developed a project plan outlining the demands of the client; in turn, they were intended to create a low burden connection between the two parties. Inventory-related information will flow in real time through this portal, including details such as quality, prices, and property descriptions.

Once renters via Airbnb demanded a house, the information would then filter back through their native framework, providing a single set of data. Next, Ferrosia recognized the need for a custom Web API to transfer data between the programs. The following details to be accepted for transition was identified by the client:
  • Property List
  • Property Detail
  • Property Pics
  • Property Availability
  • Property Rate
An implementation of an Airbnb API would then return the details of the reservation to the native system. Inventory retrieval was also included in the preliminary process, ensuring accurate display of the data at hand between the sites.

WEB API FILTERS INCOMING DATA THROUGH A 3-TIER SYSTEM

During the development process Chetu deployed the following technologies…

  • VS 2017
  • SQL Server 2012
  • .Net Framework [4.5.1]
  • C#
  • Web API
  • Log4net – For logging and tracing in application.
Using those combined technologies Chetu built a 3-tier system comprised of a Web API layer, Business Tier, and Data Tier. The Web API is the top layer of the application and exists as a filter. The Business Tier receives any requests from the Web API layer, processes them, and then communicates with the data layer. The Data Tier is responsible for communication with the client's database following the requests received from the Web and Business Tiers. Chetu chose two libraries to leverage within the project, Elmah and Log4net. Elmah was implemented to capture all the rental exceptions and display user-friendly messages regarding the exceptions within the UI. Log4net was used for logging and tracing information, warnings, and errors. Chetu formulated a deliverable for the client that had the dual-functionality they desired, preserving the integrity of their legacy system while reaping the benefits of Airbnb's global consumer following.
